  104.      * current parse environment. 
  105.      * Returns false when the buffer is empty, or when there is an error. 
  106.      * 
  107.      * This function is called repeatedly until the entire document is 
  108.      * parsed. 
  109.      * 
  110.      * This parser is most similar to a recursive descent parser. Single 
  111.      * functions represent discrete grammatical rules for the language, and 
  112.      * they are able to capture the text that represents those rules. 
  113.      * 
  114.      * Consider the function lessc::keyword(). (all parse functions are 
  115.      * structured the same) 
  116.      * 
  117.      * The function takes a single reference argument. When calling the 
  118.      * function it will attempt to match a keyword on the head of the buffer. 
  119.      * If it is successful, it will place the keyword in the referenced 
  120.      * argument, advance the position in the buffer, and return true. If it 
  121.      * fails then it won't advance the buffer and it will return false. 
  122.      * 
  123.      * All of these parse functions are powered by lessc::match(), which behaves 
  124.      * the same way, but takes a literal regular expression. Sometimes it is 
  125.      * more convenient to use match instead of creating a new function. 
  126.      * 
  127.      * Because of the format of the functions, to parse an entire string of 
  128.      * grammatical rules, you can chain them together using &&. 
  129.      * 
  130.      * But, if some of the rules in the chain succeed before one fails, then 
  131.      * the buffer position will be left at an invalid state. In order to 
  132.      * avoid this, lessc::seek() is used to remember and set buffer positions. 
  133.      * 
  134.      * Before parsing a chain, use $s = $this->seek() to remember the current 
  135.      * position into $s. Then if a chain fails, use $this->seek($s) to 
  136.      * go back where we started.
