  78.      * current parse environment. 
  79.      * 
  80.      * Returns false when the buffer is empty, or when there is an error. 
  81.      * 
  82.      * This function is called repeatedly until the entire document is 
  83.      * parsed. 
  84.      * 
  85.      * This parser is most similar to a recursive descent parser. Single 
  86.      * functions represent discrete grammatical rules for the language, and 
  87.      * they are able to capture the text that represents those rules. 
  88.      * 
  89.      * Consider the function scssc::keyword(). (All parse functions are 
  90.      * structured the same.) 
  91.      * 
  92.      * The function takes a single reference argument. When calling the 
  93.      * function it will attempt to match a keyword on the head of the buffer. 
  94.      * If it is successful, it will place the keyword in the referenced 
  95.      * argument, advance the position in the buffer, and return true. If it 
  96.      * fails then it won't advance the buffer and it will return false. 
  97.      * 
  98.      * All of these parse functions are powered by scssc::match(), which behaves 
  99.      * the same way, but takes a literal regular expression. Sometimes it is 
  100.      * more convenient to use match instead of creating a new function.

  102.      * Because of the format of the functions, to parse an entire string of 
  103.      * grammatical rules, you can chain them together using &&.

  633.     protected function expHelper($lhs, $minP) { 
  634.         $opstr = self::$operatorStr; 
  635.
