xprofile_insert_field

Insert or update an xprofile field.

Description

(bool|int) xprofile_insert_field( (string) $args = '' ); 

Returns (bool|int)

False on failure, ID of new field on success.

Parameters (1)

0. $args — Optional. (string) => ''
Array of arguments.

Options

  • List
  • Code
  • Close
  • field_id (int) => 0

    Pass the ID of an existing field to edit that field.

  • field_group_id (int) => 0

    ID of the associated field group.

  • parent_id (int) => 0

    ID of the parent field.

  • type (string) => ''

    Field type. Checked against a field_types whitelist.

  • name (string) => ''

    Name of the new field.

  • description (string) => ''

    Descriptive text for the field.

  • is_required (bool) => null

    Whether users must provide a value for the field. Default: false.

  • can_delete (bool) => null

    Whether admins can delete this field in the Dashboard interface. Generally this is false only for the Name field, which is required throughout BP. Default: true.

  • order_by (string) => ''

    For field types that support options (such as radio), this flag determines whether the sort order of the options will be default (order created) or custom..

  • is_default_option (bool) => null

    For the option field type, setting this value to true means that it'll be the default value for the parent field when the user has not yet overridden. Default: true.
